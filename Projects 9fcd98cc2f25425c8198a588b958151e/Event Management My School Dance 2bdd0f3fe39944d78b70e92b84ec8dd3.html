Overview

Primary directive for this project was to move quickly and get into development as soon as possible. With the simplicity behind the project it was doable with careful research to ensure we didn’t need to backtrack any of the user flows or journey.

Design Process

Research & Define Problems

The client and I worked to build out a user flow diagram to outline the possible outcomes of scanning a ticket. Main concerns were allowing scanners to circumvent what could be hard locked conditions. They didn't want people unable to enter events if the user ran into problems, rather the more important thing was to make sure and log every event to be reviewed later.

Strategy & Analysis

  • Collaborative effort with client input and provided wireframes.
  • Emphasis on simplicity, large buttons, and clear text for user comfort.
  • Early feedback prompted the addition of flexibility features for ticket management.

Decisions

  • Implemented a workflow to unblock and block tickets, providing users with enhanced flexibility.
  • Added features to prevent ticket misuse, allowing one ticket per student.
  • Inclusion of a student ID picture for further security and verification.

Test & Iterate

Results:

  • Enhanced flexibility and confidence for ticketing desk management.
  • Positive client response to ease of use, large buttons, and added flexibility.

Key Features:

  • Unique database integration for checking tickets against a list of students.
  • Inclusion of student ID pictures for visual verification.

User Experience:

  • Empowered users managing the ticketing desk with an intuitive and efficient system.

Lessons Learned/Final Thoughts

MySchoolDance successfully addressed the challenges posed by high school dance ticketing, offering a secure, flexible, and user-friendly solution. The addition of features such as database integration and student ID pictures contributed to a positive user experience and increased security.
